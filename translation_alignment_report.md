## Por Que Estão Desalinhados e Como Alinhar?
### Motivo Principal:
1. **Tradução em Lotes Fragmentados**: Historicamente, os idiomas **ES** (Espanhol) e **PT** (Português) receberam rodadas adicionais de tradução manual ou automática para cobrir novas telas como Settings, Quota, e Diff Viewer (atingindo ~76% de cobertura).
2. **Falta de Atualização nos Outros Idiomas**: Os outros 11 idiomas (como Alemão, Francês, Japonês, Chinês, Árabe, etc.) pararam na primeira grande leva de tradução que cobria apenas cerca de 503 chaves (~41% de cobertura).
3. **Novas Chaves Adicionadas Recentes**: Conforme novos recursos foram sendo codificados (como LaTeX, Quota, painel de terminal, etc.), as novas chaves correspondentes no `englishTemplate` não foram enviadas para tradução nos idiomas não-ingleses.

### Como Alinhar?
Podemos escrever um script Python que use uma API de Tradução (ou que gere blocos de requisição para um LLM traduzir em lotes eficientes) para traduzir as chaves faltantes de cada idioma e preencher o `arb_strings.dart` de uma vez só, garantindo alinhamento de 100% de cobertura.
